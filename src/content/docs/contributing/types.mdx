## Contribute to Spryker

There are several ways you can contribute directly to the Spryker codebase:

<CardGrid>

  <LinkCard
    title="B2B Demo Marketplace"
    description="The full Spryker demo shop — report bugs, submit PRs, or explore the codebase."
    href="https://github.com/spryker-shop/b2b-demo-marketplace"
  />

  <LinkCard
    title="Spryker Core Modules"
    description="Individual core modules — find the specific module you want to contribute to."
    href="https://github.com/spryker"
  />

  <LinkCard
    title="Report a Bug"
    description="Found something broken? Open an issue with steps to reproduce and help us fix it."
    href="https://github.com/spryker-shop/b2b-demo-marketplace/issues"
  />

  <LinkCard
    title="Submit a Pull Request"
    description="Fix bugs, add features, or improve code. PRs get fast reviews from the Spryker core team."
    href="https://github.com/spryker-shop/b2b-demo-marketplace/pulls"
  />

  <LinkCard
    title="Community Extensions"
    description="Build and share your own Spryker modules as MIT-licensed packages for the ecosystem."
    href="https://github.com/spryker-community"
  />

</CardGrid>

<Aside type="tip">
  New to open source contributions? The [B2B Demo Marketplace](https://github.com/spryker-shop/b2b-demo-marketplace) is the best starting point. For individual core modules, browse the [spryker](https://github.com/spryker) GitHub organization. Even small fixes like typos or documentation improvements are valuable!
</Aside>

## Want to contribute to our community documentation?

<Aside type="tip">
  You can easily edit this documentation on GitHub. Simply click the "Edit page" link that is shown at the bottom of each page to make changes directly on GitHub. If it's your first time doing this, you can find more information on how to contribute in our [Content Contributions guide](https://github.com/spryker-community/spryker-community.github.io?tab=readme-ov-file#contributing).
</Aside>

It's a community effort to keep our documentation up to date and accurate. If you find an error or have a suggestion for improvement, we encourage you to contribute. Your feedback helps us maintain the highest quality documentation for our users. 🙏
